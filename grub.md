# GRUB Custom Boot Order Setup

This document records the steps and scripts used to force GRUB to place the Windows Boot Manager exactly between "Arch Linux" and the "Advanced options for Arch Linux" submenu.

## 1. The Reorder Script

We use a Python script to parse the generated `grub.cfg` and move the `os-prober` block (Windows) right before the Advanced Options submenu.

Create the file `/usr/local/bin/grub-reorder.py`:

```bash
sudo nano /usr/local/bin/grub-reorder.py
```

Paste the following content exactly:

```python
#!/usr/bin/env python3
import os

CFG = "/boot/grub/grub.cfg"

# Read the generated grub config
with open(CFG, 'r') as f:
    text = f.read()

# INJECT CLASSES so the Matrix theme can find your images!
text = text.replace("submenu 'Advanced options for ", "submenu --class adv_arch 'Advanced options for ")
text = text.replace("menuentry 'UEFI Firmware Settings'", "menuentry --class uefi 'UEFI Firmware Settings'")

# Locate the Windows block generated by os-prober
os_prober_start = text.find("### BEGIN /etc/grub.d/30_os-prober ###")
os_prober_end = text.find("### END /etc/grub.d/30_os-prober ###")

if os_prober_start != -1 and os_prober_end != -1:
    os_prober_end += len("### END /etc/grub.d/30_os-prober ###")
    os_prober_block = text[os_prober_start:os_prober_end]
    
    # Remove the Windows block from the bottom
    text = text[:os_prober_start] + text[os_prober_end:]
    
    # Locate the start of the Advanced Options submenu (now with the injected class)
    submenu_start = text.find("submenu --class adv_arch 'Advanced options for ")
    
    if submenu_start != -1:
        # Insert the Windows block right between Arch Linux and Advanced Options
        text = text[:submenu_start] + os_prober_block + "\n" + text[submenu_start:]
        
        # Save the reordered config
        with open(CFG, 'w') as f:
            f.write(text)
        print("=> GRUB menu successfully reordered and classes injected: 1. Arch 2. Windows 3. Advanced 4. UEFI")
    else:
        # Save config anyway to keep the class injections
        with open(CFG, 'w') as f:
            f.write(text)
        print("=> Could not find Advanced Options submenu, skipping reorder. Classes were still injected.")
else:
    # Save config anyway to keep the class injections
    with open(CFG, 'w') as f:
        f.write(text)
    print("=> Could not find Windows os-prober block, skipping reorder. Classes were still injected.")
```

Make the script executable:

```bash
sudo chmod +x /usr/local/bin/grub-reorder.py
```

## 2. The Pacman Hook

To make sure this script runs automatically every time the kernel or GRUB is updated, we use a pacman hook.

Create the hook file:

```bash
sudo mkdir -p /etc/pacman.d/hooks
sudo nano /etc/pacman.d/hooks/zz-grub-reorder.hook
```

Paste the following content:

```ini
[Trigger]
Operation = Install
Operation = Upgrade
Operation = Remove
Type = Package
Target = linux
Target = linux-lts
Target = linux-zen
Target = grub

[Action]
Description = Reordering GRUB boot menu to place Windows 2nd...
Depends = python
When = PostTransaction
Exec = /usr/local/bin/grub-reorder.py
```

## 3. Applying and Testing

To test the setup instantly (without waiting for a kernel update), you run:

```bash
# Generate the default grub configuration
sudo grub-mkconfig -o /boot/grub/grub.cfg

# Run our custom reorder script
sudo /usr/local/bin/grub-reorder.py
```

Once run, you will see the success message, and on your next boot, your menu order will be:
1. Arch Linux
2. Windows Boot Manager
3. Advanced options for Arch Linux
4. UEFI Firmware Settings

### Troubleshooting: Blank Screen / Resolution Issue

If you reboot and cannot see anything (or the screen is blank/unresponsive), it is likely a resolution mismatch between GRUB and your monitor's capabilities. 

**To quickly fix the GRUB resolution, run these commands in your terminal:**

```bash
# 1. Update the GRUB resolution and force graphical output
sudo python3 -c "
with open('/etc/default/grub', 'r') as f: text = f.read()
text = text.replace('GRUB_GFXMODE=1280x1024x32,auto', 'GRUB_GFXMODE=1920x1080x32,auto\nGRUB_TERMINAL_OUTPUT=gfxterm')
with open('/etc/default/grub', 'w') as f: f.write(text)
"

# 2. Regenerate the GRUB configuration file so the changes take effect
sudo grub-mkconfig -o /boot/grub/grub.cfg

# 3. Re-run your custom boot order script to keep Windows in the middle
sudo /usr/local/bin/grub-reorder.py
```

*Note: Replace `1920x1080x32` with your monitor's native resolution if it is different.*
